Backend Management System - FUNCAVIDA

Project Description

The project implements a RESTful API with layered architecture (MVC + Service/Repository), integrating JWT authentication, role-based permission management, and a complete password recovery system via email. Developed in a team following agile methodologies and collaborative development practices with version control.

GitHub Repository

View on GitHub

Project Architecture

Backend architecture based on Node.js with MVC pattern (Model-View-Controller), implementing RESTful APIs for communication with the frontend and external services:

  • RESTful API with Express.js for routing and controller handling
  • Email sending service with Nodemailer for password recovery
  • Database management and data persistence with Prisma ORM and MySQL
  • User authentication and authorization system with JWT and bcrypt
  • Integration with NGO management modules:
    • User, Role, and Permission Management (RBAC - Role-Based Access Control)
    • Survivor, Sponsor, and Volunteer Management (With complex relationships)
    • Activity, Venue, and Asset Management (Complete events and resources system)
    • Supplier and Category Management (Inventory control and classification)
    • Emergency Contact System (For all participants)
    • Security Logs (Audit logs for all critical operations)
    • Password Recovery (Complete system with secure tokens and expiration)
    • Statistics and Reports (Dashboard with key metrics of the organization)

Technologies Applied

  • Node.js - JavaScript runtime environment for server-side development
  • Express.js - Minimalist and flexible web framework for Node.js
  • Nodemailer - Library for sending emails (password recovery)
  • JavaScript - Main programming language for the backend
  • Prisma ORM - Modern ORM for Node.js with TypeScript support
  • MySQL - Relational database management system
  • JWT (JSON Web Tokens) - Secure stateless authentication and authorization
  • bcrypt - Password hashing with strong encryption algorithm
  • Git - Distributed version control for collaborative development

Frontend Management System - FUNCAVIDA

Comprehensive web management system for a Costa Rican foundation

Project Description

Frontend system developed for FUNCAVIDA (Cancer Life Foundation), a Costa Rican organization dedicated to supporting cancer patients. The web application enables comprehensive management of cancer survivors, sponsors, volunteers, venues, assets, categories, and emergency contacts.

The system includes advanced features such as JWT authentication, granular permission system by window, bulk data export to Excel with corporate styles, statistics system with interactive visualizations (Chart.js), secure session management, and reusable modals for success/error messages. It comes with seeders to generate test data and complete technical documentation for all modules.

GitHub Repository

View on GitHub

Project Architecture

MVC Frontend Architecture with ES6 modules: Single Page Application (SPA) that consumes an external REST API. Uses a responsibility separation pattern with an organized folder structure in View, Controller, and Core.

  • View/ - HTML views organized by module (User, Cancer, GodParent, Headquarter, Assets, Category, EmergencyContact, Survivor, Volunteer)
  • Controller/ - Business logic in modular JavaScript:
    • api{Module}.js - HTTP client for communication with REST API using fetch
    • index{Module}.js - Module orchestration and event handling
    • list{Module}.js - List rendering with dynamic filters
    • form{Module}.js - Validation and form submission
    • utils/ - Shared utilities (JWT, permissions, modals, Excel export, API client)
  • Core/ - Authentication middleware and centralized error handling
  • Content/ - Unified CSS styles with CSS variables and responsive design
  • seeders/ - Test data generators for sponsors and venues
  • docs/ - Detailed technical documentation for each module and utility

Technologies Applied

  • JavaScript ES6+ - Native modules, async/await, destructuring
  • HTML5 - Semantic and accessible
  • CSS3 - Custom Properties, Grid, Flexbox
  • jQuery 3.7.1 - DOM manipulation and AJAX
  • Chart.js 4.4.0 - Interactive charts (bar, line, pie)
  • xlsx-js-style 1.2.0 - Excel export with corporate styles
  • Express.js 4.19.2 - Static file server
  • JSON Web Tokens (JWT) - Secure authentication and authorization
  • Fetch API - Communication with REST API
  • Git - Distributed version control

IntelliVerse

Multiplatform artificial intelligence platform

Project Description

IntelliVerse is a multiplatform artificial intelligence platform that integrates multiple AI models (Gemini, Groq, Mistral, HuggingFace) to offer a suite of intelligent applications: personalized diet tracker, one-click travel planner, text style translator, educational assistant, and decision-making system, all with modular architecture and parallel execution of models.

GitHub Repository

View on GitHub

Project Architecture

Hybrid full-stack architecture with Python backend and native Android mobile applications, implementing a multi-model AI system with automatic failover:

  • Flask backend with RESTful API for AI model orchestration
  • Multi-model system with parallel execution (RACE, FALLBACK, ALL strategies)
  • Integration of 5 AI providers: Gemini, Groq, Mistral, OpenRouter, HuggingFace
  • Native Android applications with Kotlin and Jetpack Compose
  • Modular architecture with 6 independent specialized applications
  • Dynamic prompt management with customizable JSON templates
  • Centralized configuration management and fault tolerance
  • Dockerfile and Docker Compose for containerized deployment

Technologies Applied

  • Python & Flask - Modular backend RESTful with blueprints
  • Kotlin - Native Android development with type-safety
  • Jetpack Compose - Modern declarative UI for Android
  • Google Gemini API - Google's multimodal AI model
  • Groq & Mistral AI - High-performance LLM models
  • HuggingFace - Access to open-source ML models
  • Gradle & KTS - Multi-module build system for Android
  • Room Database - Local persistence with SQLite
  • Dagger-Hilt - Dependency injection in Android
  • Docker - Containerization and deployment of the backend
  • Git - Version control with mono-repo structure

Minesweeper with React

Modern implementation of the classic Minesweeper game

Project Description

Reimplementation of the iconic Minesweeper game using React with modern hooks and Redux Toolkit for state management. Includes multiple difficulty levels, real-time timer, remaining mines counter, flag system, and automatic detection of victory or defeat.

GitHub Repository

View on GitHub

Project Architecture

React application with functional components, hooks, and Redux Toolkit for global state, structured for maintainability and scalability:

  • Minesweeper - Main component that manages the game state and timer
  • Board - Component that renders the game board cells
  • Cell - Individual component for each board cell with reveal and flag logic
  • GameControls - Controls to change difficulty, restart game, and show statistics
  • WinModal - Modal that appears upon winning, showing time and difficulty
  • minesweeperSlice - Redux slice with game logic, board generation, mine placement, and recursive reveal algorithms
  • Game states: idle, playing, won, lost
  • Recursive reveal algorithm for adjacent empty areas

Technologies Applied

  • React 19 - Library for user interfaces with functional components
  • JavaScript ES6+ - Arrow functions, destructuring, modules, and async/await
  • CSS3 - Styles with CSS variables, flexbox, and animations
  • React Hooks - useState, useEffect, useSelector, useDispatch
  • Redux Toolkit - Global state management with slices and reducers
  • Vite - Fast build tool and development server
  • Git - Version control

Key Features

  • Multiple Difficulties: Easy (5x5, 5 mines), Medium (10x10, 15 mines), Hard (20x20, 60 mines)
  • Timer: Real-time countdown during the game in MM:SS format
  • Mines Counter: Visual indicator of remaining mines (total mines - placed flags)
  • Flags System: Right-click to mark suspicious cells, limited to the number of mines
  • Victory System: Automatic detection of victory when all safe cells are revealed
  • Quick Restart: Button to restart the game while keeping the current difficulty
  • Intuitive Interface: Clean design with visual feedback and status messages
  • Game Over: Automatic revelation of all mines upon losing
